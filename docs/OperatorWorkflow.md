# Работа оператора с рабочего места

## Цель
Зафиксировать модель эксплуатации, в которой оператор не работает через постоянный интерактивный вход на сервер, а управляет `AWG` со своего рабочего места.

## Принцип
Оператор работает локально.

Основной интерфейс оператора:
- локальный `awgctl`;
- локальный каталог `awg_clients/`;
- SSH alias до сервера.

Детали локального CLI и `sudoers` описаны в [docs/Awgctl.md](Awgctl.md).

## Что не считаем правильным процессом
Не считаем целевым такой сценарий:
```bash
ssh <server_alias>
sudo awg-add-client test
sudo awg-list-clients
sudo awg-export-client client01_test
sudo awg-revoke-client client01_test
```

Это допустимо для отладки и ручной диагностики, но не как постоянная схема работы оператора.

## Что считаем правильным процессом
Оператор должен работать со своего рабочего места командами уровня orchestration.

Целевой вид:
```bash
awgctl list-clients
awgctl add-client test
awgctl export-client client01_test
awgctl revoke-client client01_test
```

Локальная команда `awgctl` должна:
- вызывать на сервере нужный backend-скрипт;
- забирать результат на локальную машину;
- очищать транзитную зону после успешной передачи.

## Разделение ответственности
На сервере остаются:
- реальный `awg0.conf`;
- client registry;
- generated client configs;
- QR;
- backend-команды `awg-*`.

На рабочем месте должны появиться:
- `script_client/awgctl`;
- локальный каталог `awg_clients/`;
- SSH-конфиг/alias для доступа к серверу.

## Роль `/home/<operator_user>/export`
`/home/<operator_user>/export` остается транзитной зоной на сервере.

Правила:
- файлы попадают туда только для передачи оператору;
- держать их там постоянно не нужно;
- после успешного получения на рабочее место оператор должен очищать `export`;
- повторная выдача должна выполняться через `awg-export-client`.

## Действия оператора
1. Выполнить `awgctl list-clients`.
2. При создании нового клиента выполнить `awgctl add-client <name>`.
3. Забрать полученные файлы из `awg_clients/` и передать их пользователю.
4. При повторной выдаче выполнить `awgctl export-client <client_id>`.
5. При отзыве выполнить `awgctl revoke-client <client_id>`.

## Проверки результата этапа
- Зафиксировано, что оператор работает локально, а не через постоянный SSH-shell.
- Зафиксировано, что `awg-*` на сервере являются backend-командами.
- Зафиксировано, что `/home/<operator_user>/export` является транзитной зоной.
- Зафиксирован рабочий процесс через `awgctl`.

## Ожидаемый итог
Операторский workflow отделен от серверной реализации.
