# Подготовка чистого VPS для AWG

## Цель
Подготовить чистый VPS сразу после первого входа под `root`: проверить ОС, включить корректную `UTF-8` локаль, установить базовые инструменты и build toolchain, рано проверить установку `AmneziaWG` и вопрос с ядром, при необходимости сразу выполнить reboot, и только потом перейти к bootstrap SSH.

## Входные условия
- Есть первый рабочий доступ к серверу под `root`.
- Сервер только что создан или находится в чистом состоянии.
- Целевая ОС: `Debian 12`.
- Дальнейшая настройка будет выполняться вручную по документам из каталога `docs/`.

## Шаг 1. Первый вход на сервер
Подключиться к чистому серверу под `root`. Обычно на новых VPS первый вход доступен только по паролю.

Типовой вариант:
```bash
ssh -o PubkeyAuthentication=no -o PreferredAuthentications=password root@<server_ip>
```

Если провайдер выдает не `root`, а временного пользователя, используйте его логин:
```bash
ssh -o PubkeyAuthentication=no -o PreferredAuthentications=password <initial_user>@<server_ip>
```

## Шаг 2. Проверка ОС
Проверить, что сервер действительно запущен на `Debian 12`.

Команда:
```bash
cat /etc/os-release
```

Ожидаемый фрагмент:
```text
PRETTY_NAME="Debian GNU/Linux 12 (bookworm)"
```

## Шаг 3. Проверка локали UTF-8
Проверить локаль `UTF-8`. Она нужна для корректной работы shell, `vim`, логов и генерации конфигов.

Текущая локаль:
```bash
locale
```

Проверка на `UTF-8`:
```bash
locale | grep UTF-8
```

Если `UTF-8` не включена, задайте безопасный вариант `C.UTF-8`:
```bash
printf 'LANG=C.UTF-8\nLC_ALL=C.UTF-8\n' > /etc/default/locale
export LANG=C.UTF-8
export LC_ALL=C.UTF-8
```

Повторная проверка:
```bash
locale
```

## Шаг 4. Обновление списка пакетов
Сразу после первого входа под `root` обновляем индекс пакетов.

Команда:
```bash
apt update
```

## Шаг 5. Установка базовых пакетов и build toolchain
Сразу после первого входа под `root` установить весь набор инструментов, которого хватит и для дальнейшей настройки, и для исследования проблем установки `AmneziaWG`.

На этом этапе нужны:
- `tmux` для устойчивой работы в сессии;
- `vim` для правки файлов;
- `curl` и `wget` для загрузки файлов;
- `sudo` для дальнейшей работы административного пользователя;
- `git`, `build-essential`, `dkms`, `libmnl-dev`, `libelf-dev`, `libqrencode-dev`, `pkg-config` для установки и fallback-сборки `AmneziaWG`.

Команда:
```bash
apt install -y tmux vim curl wget sudo git build-essential dkms libmnl-dev libelf-dev libqrencode-dev pkg-config
```

Проверка:
```bash
tmux -V
vim --version | head -n 1
curl --version | head -n 1
wget --version | head -n 1
sudo -V | head -n 1
git --version
dkms --version
```

## Шаг 6. Ранняя проверка ядра и готовности к AmneziaWG
До создания пользователя и изменения `sshd` проверить, под какое ядро живет сервер и не нужен ли ранний reboot.

Команды:
```bash
uname -r
apt-cache policy linux-image-amd64 linux-headers-amd64
```

Зафиксировать:
- текущее ядро `uname -r`;
- есть ли более новый meta-package ядра;
- будет ли нужен `reboot` до продолжения настройки.

## Шаг 7. Ранняя установка AmneziaWG
Пока еще есть вход под `root` по паролю, сразу проверить пакетный сценарий `AmneziaWG`.

На этом этапе:
- подключить репозиторий `AmneziaWG`;
- установить пакет;
- проверить, собрался ли `DKMS`;
- понять, совпадает ли собранный модуль с текущим ядром.

Детальный сценарий остается в [docs/AmneziaWGInstall.md](AmneziaWGInstall.md). На этом этапе придерживаться таких правил:
- если пакетный путь требует нового ядра, reboot делаем сразу;
- если `DKMS` собрался под новое ядро, продолжаем только после reboot;
- если пакетный путь не работает даже после нормализации ядра и headers, потом идем в fallback manual build.

## Шаг 8. Ранний reboot при необходимости
Если установка `AmneziaWG` или обновление kernel/headers потребовали нового ядра, выполнить reboot сразу, а не после полного bootstrap.

Команды:
```bash
reboot
```

После входа повторно проверяем:
```bash
uname -r
dkms status
```

Ожидаемый результат:
- сервер работает уже на целевом ядре;
- модуль `amneziawg` собран или готов к загрузке именно под текущее ядро.

## Шаг 9. Запуск рабочей tmux-сессии
Чтобы не потерять сессию при обрыве SSH, вести дальнейшую настройку внутри `tmux`.

Создание сессии:
```bash
tmux new -s awg-setup
```

Отключение от сессии:
```bash
Ctrl+b d
```

Повторное подключение:
```bash
tmux attach -t awg-setup
```

## Что на этом этапе не делайте
- не создавайте административного пользователя;
- не меняйте `sshd_config`;
- не настраивайте firewall;
- не создавайте `/etc/amnezia` и `/var/lib/amnezia`;
- не создавайте клиентские или серверные конфиги.

Эти действия идут отдельными следующими этапами.

## Проверки результата этапа
- `cat /etc/os-release` показывает `Debian 12`.
- `locale` показывает локаль с `UTF-8`.
- `tmux`, `vim`, `curl`, `wget`, `sudo`, `git`, `dkms` установлены и запускаются.
- До дальнейшего bootstrap уже проверен сценарий `AmneziaWG` и понятен вопрос с ядром/reboot.

## Ожидаемый итог
После этого этапа сервер готов к bootstrap:
- базовое окружение готово;
- локаль корректна;
- базовые пакеты и toolchain уже установлены до изменения `sshd`;
- вопрос с kernel/headers/`AmneziaWG` решается до создания операционного пользователя;
- ручная работа может продолжаться в `tmux`.
